기존 **ASP 기반 웹사이트**의 프론트엔드를 **Next.js**로 리뉴얼하고 있었습니다.

영상 콘텐츠 중심의 사이트이기 때문에, 가장 중요한 미션 중 하나는 **"기존 ASP가 어떻게 영상을 보여줬는지"** 파악하고, 이를 **Next.js 환경에서 어떻게 자연스럽고 효율적으로 재현할까?**

## ASP에서 영상을 어떻게 처리하고 있었나

![](/images/optimizing-video-streaming/asp-m3u8.png)

네트워크 탭을 열어보니 `.m3u8` 파일이 두 개 들어오고, 그 밑으로 `.ts` 세그먼트들이 쭉 들어오는 걸 확인할 수 있었습니다.

`.m3u8`, `.ts` 파일의 URL을 열어보면 **Wowza** 서버로 되어 있습니다.

```
https://wowza....../vod/_definst_/mp4:0000565/AAAAAA_720p.mp4/playlist.m3u8
```

<details>
<summary>**Wowza란?**</summary>

Wowza는 업로드된 MP4 파일을 실시간으로 **HLS(HTTP Live Streaming)** 포맷으로 변환하여 제공하는 스트리밍 서버입니다.

따로 인코딩할 필요 없이, `MP4`만 서버에 올려두면 `.m3u8` (플레이리스트)와 `.ts` (세그먼트) 파일을 자동으로 생성해서 클라이언트에 전달합니다.

</details>

<details>
<summary>**Streaming 비교 with GPT**</summary>

| 방식            | 서비스 예시            | 인코딩 필요          | 커스텀        | 특징               |
| --------------- | ---------------------- | -------------------- | ------------- | ------------------ |
| Wowza 서버      | Wowza Streaming Engine | 불필요 (실시간 변환) | 가능          | 안정적, 비용 고정  |
| 매니지드 플랫폼 | AWS IVS, Mux           | 플랫폼이 관리        | API로 가능    | 편리, 유료         |
| CDN + VOD       | S3 + CloudFront        | 중간쯤 (파일 관리만) | 가능          | 매우 저렴, 안정적  |
| iframe 임베딩   | YouTube, Vimeo         | 거의 안 함           | 불가능/제한적 | 공짜지만 통제 불가 |

&nbsp;

**VOD란?**

**Video On Demand** (주문형 영상)으로, 넷플릭스처럼 사용자가 요청할 때 영상을 "다운받는 것처럼 보여주지만 실은 스트리밍"하는 방식입니다.
실시간 라이브가 아닌 영상 재생은 거의 다 VOD라고 보면 됩니다.

&nbsp;

**CDN VOD이란?**

📦 CDN + VOD = “파일을 미리 올려두고 스트리밍처럼 제공”

- 영상 같은 큰 파일을 전 세계 서버에 **미리 분산**시켜서 사용자에게 빠르게 제공
- `https://cdn.example.com/videos/intro.mp4` 이런 주소로 빠르게 재생됨

&nbsp;

**📦 CDN + VOD 방식은 어떤 "스트리밍"?**

- CDN에서 mp4 또는 HLS 파일을 **조각내서 조금씩 다운로드하며 재생**
- 사용자가 요청한 시점의 영상만 받아오고, 나머지는 필요할 때 받아옴
- 즉, 넷플릭스나 유튜브처럼 작동함
  📌 **이건 On-Demand Streaming (VOD 스트리밍)** 이라고 부릅니다.

| 구분          | CDN + VOD                | Wowza, IVS                     |
| ------------- | ------------------------ | ------------------------------ |
| 스트리밍 타입 | 🎬 On-Demand (요청 기반) | 🔴 Live (실시간 방송)          |
| 데이터 원본   | 미리 업로드된 파일       | 현재 송출 중인 비디오          |
| 지연 시간     | 없음 또는 3~10초         | 1~5초 (HLS), 1초 이하 (WebRTC) |
| 플레이어      | `<video>` / `hls.js` 등  | `hls.js`, WebRTC 등            |
| 업로드 방식   | 수동 (S3, Firebase 등)   | 인코더 → RTMP Push             |
| 비용 구조     | 저장 용량 + CDN 트래픽   | 송출 시간, 시청자 수           |

&nbsp;

**Wowza Engine vs Wowza Cloud vs AWS IVS 비교**

| 기능           | Wowza Streaming Engine | Wowza Cloud            | AWS IVS     |
| -------------- | ---------------------- | ---------------------- | ----------- |
| 방식           | 직접 설치, 내 서버     | Wowza 운영             | AWS 운영    |
| Live Streaming | ✅                     | ✅                     | ✅          |
| VOD 지원       | ✅ (mp4 → HLS)         | ❌ 직접 저장 구현 필요 | ❌          |
| 유연성         | 매우 높음              | 보통                   | 낮음        |
| 관리 편의      | 낮음 (내가 관리)       | 높음                   | 매우 높음   |
| 비용           | 설치형/고정            | 구독 or 사용량         | 사용량 기준 |

</details>

## 그럼 클라이언트에서 어떻게 처리할까

서버에서는 똑같이 **Wowza Streaming Engine**를 이용해서 `.mp4`를 **HLS**로 변환해서 `.m3u8 URL`로 넘겨줌
Next.js에서는 `.m3u8 URL`를 **Video.js**로 비디오를 렌더링하기로 결정

### ASP에서 Nextjs 환경으로의 처리 흐름

기존 ASP에서 흐름

```html
[사용자 요청] ↓ [ASP 서버] ↓ [Wowza로 전달: mp4 → HLS 변환 처리] ↓ ASP가
<video src="...m3u8">
  포함된 HTML 렌더링 ↓ 브라우저가 HLS 재생 (Safari: 네이티브, 그 외: HLS.js or
  video.js)
</video>
```

Next.js에서 흐름

```html
[사용자 요청] ↓ [Next.js SSR or SSG 페이지 로딩] ↓ (초기 렌더 중) → API 요청으로
영상 재생용 데이터(m3u8 URL 등) 비동기 fetch ↓ Client-Side에서 video.js 초기화
(useEffect 등에서) ↓ video.js가 m3u8 받아 파싱하고 ts 조각 fetch 요청 ↓ Media
Source Extensions로 <video>에 연결 → 재생</video>
```

### Videojs가 m3u8를 어떻게 다루는가

동일한 Wowza 서버 구조라면 video.js도 동일한 스트리밍 네트워크 트래픽을 발생시킴 (위 이미지와 같음)

- **비디오 콘텐츠의 네트워크 요청 구조는 동일**
  (video.js는 단지 UI와 이벤트 핸들링 + 호환성 해결용 레이어일 뿐)

> 클라이언트에서 해줘야 하는 것은 결국 비디오 영상에 대한 렌더링 최적화를 해주면 될듯하다.

VideoJS 공홈에 예시 코드가 잘 나와있고 커스텀도 자유롭게 할 수 있다.

```jsx
import React from 'react';
import videojs from 'video.js';
import 'video.js/dist/video-js.css';

export const VideoJS = (props) => {
  const videoRef = React.useRef(null);
  const playerRef = React.useRef(null);
  const { options, onReady } = props;

  React.useEffect(() => {
    // Make sure Video.js player is only initialized once
    if (!playerRef.current) {
      // The Video.js player needs to be _inside_ the component el for React 18 Strict Mode.
      const videoElement = document.createElement('video-js');

      videoElement.classList.add('vjs-big-play-centered');
      videoRef.current.appendChild(videoElement);

      const player = (playerRef.current = videojs(videoElement, options, () => {
        videojs.log('player is ready');
        onReady && onReady(player);
      }));

      // You could update an existing player in the `else` block here
      // on prop change, for example:
    } else {
      const player = playerRef.current;

      player.autoplay(options.autoplay);
      player.src(options.sources);
    }
  }, [options, videoRef]);

  // Dispose the Video.js player when the functional component unmounts
  React.useEffect(() => {
    const player = playerRef.current;

    return () => {
      if (player && !player.isDisposed()) {
        player.dispose();
        playerRef.current = null;
      }
    };
  }, [playerRef]);

  return (
    <div data-vjs-player>
      <div ref={videoRef} />
    </div>
  );
};

export default VideoJS;
```

```jsx
import React from 'react';

// This imports the functional component from the previous sample.
import VideoJS from './VideoJS';

const App = () => {
  const playerRef = React.useRef(null);

  const videoJsOptions = {
    autoplay: true,
    controls: true,
    responsive: true,
    fluid: true,
    sources: [
      {
        src: '/path/to/video.mp4',
        type: 'video/mp4',
      },
    ],
  };

  const handlePlayerReady = (player) => {
    playerRef.current = player;

    // You can handle player events here, for example:
    player.on('waiting', () => {
      videojs.log('player is waiting');
    });

    player.on('dispose', () => {
      videojs.log('player will dispose');
    });
  };

  return (
    <>
      <div>Rest of app here</div>
      <VideoJS options={videoJsOptions} onReady={handlePlayerReady} />
      <div>Rest of app here</div>
    </>
  );
};
```
